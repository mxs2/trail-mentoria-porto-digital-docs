# Governança e LGPD — Projeto Trail

Este documento define as **regras obrigatórias de governança, ética e proteção de dados**
para o Projeto Trail, no contexto da mentoria Porto Digital.

📌 O cumprimento destas regras é **obrigatório para todos os participantes**.

---

## 🎯 Objetivo deste Documento

- Garantir uso seguro de um **repositório público**
- Proteger alunos, mentores e instituições
- Evitar exposição de dados sensíveis
- Ensinar boas práticas profissionais de governança desde o início

---

## 🌐 Natureza do Repositório

Este repositório é **PÚBLICO**.

Isso significa que:
- Qualquer pessoa pode visualizar o conteúdo
- O histórico de alterações é permanente
- O conteúdo pode ser indexado por mecanismos de busca

📌 **Portanto, todo conteúdo aqui publicado deve ser tratado como público por padrão.**

---

## 📄 Fonte Única da Verdade

> O **Documento de Briefing / Discovery** é a fonte única da verdade do projeto.

Regras:
- Todo artefato deve derivar do briefing
- Ajustes ou interpretações devem ser **explicitamente justificados**
- O briefing **não deve ser sobrescrito**, apenas interpretado

---

## 🔐 LGPD — Regras Obrigatórias

É **estritamente proibido** versionar neste repositório:

### 🚫 Dados Pessoais Reais
- Nome completo de pessoas reais
- E-mails pessoais ou corporativos
- Telefones
- CPF, RG, matrícula acadêmica
- Fotos, vídeos ou áudios identificáveis

### 🚫 Informações Sensíveis
- Credenciais (tokens, senhas, chaves)
- URLs privadas
- Dados de performance individual identificável
- Informações internas de sistemas reais

---

## ✅ O que é PERMITIDO

- Dados **fictícios**
- Dados **anonimizados**
- Exemplos genéricos
- Estruturas sintéticas para fins didáticos

📌 Exemplo aceitável:
> “Estudante A”, “Mentor X”, “Empresa Parceira”

📌 Exemplo proibido:
> Nome real + desempenho + feedback individual

---

## 🧠 Princípio de Segurança do Projeto

> **Compliance by Design**

A segurança não é uma etapa final.
Ela faz parte do design desde o início.

Todo participante deve se perguntar:
> “Esse conteúdo poderia gerar risco se visto fora do contexto da mentoria?”

Se a resposta for “talvez”, **não publique**.

---

## 📦 Regras de Entregas e Versionamento

- Todo conteúdo oficial deve estar em **Markdown (.md)**
- Um arquivo por encontro (salvo exceções)
- Entregas devem ser claras e autoexplicativas
- Nada deve existir apenas “de forma verbal”

📌 O repositório é a **memória oficial do projeto**.

---

## 🚫 O que NÃO é permitido no repositório

- Prints de telas com dados reais
- Dumps de banco de dados
- Exportações de ferramentas reais
- Conteúdo ofensivo, discriminatório ou antiético
- Uso do repositório para fins pessoais

---

## ⚖️ Responsabilidade dos Participantes

Cada participante é responsável por:
- Ler este documento
- Seguir as regras
- Proteger dados de terceiros
- Alertar o mentor caso identifique risco

O desconhecimento das regras **não isenta responsabilidade**.

---

## 🧭 Papel do Mentor

O mentor:
- Orienta tecnicamente
- Garante aderência às regras
- Pode solicitar ajustes ou remoções de conteúdo
- Atua como guardião da governança do projeto

---

## ✅ Consequências do Não Cumprimento

O descumprimento das regras pode resultar em:
- Solicitação de correção imediata
- Remoção de conteúdo do repositório
- Desconsideração da entrega para fins de avaliação
- Encaminhamento para coordenação do programa (em casos graves)

---

## 🔁 Revisão deste Documento

Este documento pode evoluir ao longo do projeto, desde que:
- Mudanças sejam discutidas
- Versionadas
- Justificadas

---

> **Lembrete final:**  
> Projetos profissionais não são apenas sobre código.  
> São sobre responsabilidade, ética e confiança.
